# 初学者算法学习页与 GitHub Pages 设计

## 目标

把现有的 60 道算法、模型手写和 ASR 面试题改造成适合基础遗忘者自学的静态网站，并通过 GitHub Actions 自动发布到 GitHub Pages。

## 用户与成功标准

用户能从每题的通俗问题描述开始，理解所需基础概念、算法推导、可运行的 Python 代码和每个追问的答案。推送到 `main` 后，公开页面自动更新。

## 内容模型

每道题都必须包含以下必填内容：

1. `beginnerSummary`：不用术语的题目解释与目标。
2. `prerequisites`：本题前必须理解的概念、变量或数据结构。
3. `workedExample`：最小输入的逐步演练。
4. `derivation`：从朴素思路到最优解的推导。
5. `code`：可直接复制运行的完整 Python 函数；必要的导入、辅助类型和边界处理不能省略。
6. `lineByLine`：按代码片段解释语句目的及原因。
7. `complexity`、`edgeCases`、`pitfalls`：复杂度、边界和常见错误。
8. `followUps`：每个追问包含问题与对应完整答案，而不是两个依赖下标对齐的数组。

所有 60 道题均满足上述结构；题库校验会拒绝缺项、空数组及无对应答案的追问。

## 页面体验

复习模式默认使用“分步引导”顺序呈现：通俗解释、概念、手动演练、推导、完整代码、逐行解释、复杂度与追问。追问使用原生可访问的可展开问答控件，减少首屏认知负担。

模拟模式保持答案默认隐藏。用户可先独立作答，再按学习顺序逐段揭晓内容；标记掌握与本地进度保存继续保留。

## 工程边界

网站继续保持无构建步骤的 HTML/CSS/ES Modules 静态架构。题目数据可拆分为专门的内容模块，渲染和校验逻辑保留在独立模块，避免 `app.js` 继续承担内容编写职责。

为内容校验和渲染段落顺序添加 Node 内置测试运行器测试。测试不执行题目中的 Python，而是验证内容契约、追问问答配对和页面可用的结构数据。

## 发布

新增 GitHub Actions 工作流：在 `main` 分支的推送及手动触发时，上传仓库根目录为 GitHub Pages 制品并部署。仓库的 Pages Source 设为 GitHub Actions；首次启用后页面地址为 `https://zhitao-zeng.github.io/algorithm-interview-prep/`。

## 非目标

本次不引入后端、登录、云端进度同步、题目编辑后台或第三方 UI 框架。不会修改用户已有的题目分类和模拟面试计时规则。

## 验收

- 所有 60 题通过内容完整性校验。
- 每题都有可运行风格的完整 Python 代码及逐行解释。
- 每个追问均在页面中显示对应答案。
- 单元测试与 JavaScript 语法检查通过。
- 推送 `main` 后，GitHub Actions 自动部署 Pages。
